# Cognex Scanner Test (康耐视扫码枪测试工具)

这是一个基于 **C# WinForms** 开发的康耐视（Cognex）扫码枪测试工具。与常见的网络 Socket 字符触发不同，本项目专为工业自动化场景设计，采用 **Modbus TCP 协议** 实现了对康耐视扫码器的寄存器级控制、拍照触发及条码数据解析。

---

## 🛠️ 功能特点

* **Modbus TCP 核心驱动**：基于 `NModbus` 开源库，通过读写控制扫码枪的输入寄存器与线圈[cite: 2, 4]。
* **寄存器级控制逻辑**：
  * **线圈触发**：向 `0` 号线圈（`TriggerCoil`）写入 `true` 触发扫码器拍照。
  * **状态自动复位**：等待解码延迟后自动复位线圈为 `false`，准备下一次触发。
  * **头部信息校验**：自动读取从 `2000` 起始地址开始的 5 个输入寄存器，根据状态码校验读码是否成功，并获取条码数据的实际字节长度。
* **大端序字节流解析**：内部自动处理 Modbus 16位寄存器的大端序（High Byte First）数据转换，精准还原为标准的 ASCII 码条码字符串[cite: 2]。
* **健壮的通信机制**：
  * 具备 **TCP 异步连接超时** 保护（默认 2000ms），避免因现场网络未连通而导致主界面卡死[cite: 2]。
  * 采用 `InvokeRequired` 检查并处理跨线程日志追加，确保 UI 线程在频繁读码与回调时的交互安全。
## 💻 开发与运行环境

- **开发工具**：Microsoft Visual Studio 2019 或更高版本
    
- **目标框架**：.NET Framework
    
- **核心依赖**：`NModbus` (可通过 NuGet 包管理器安装)
    
- **硬件对接**：支持 Modbus TCP 协议的 Cognex DataMan 系列扫码枪

## 🚀 快速上手指南

### 1. 扫码枪硬件配置

在使用本工具前，需要通过 **DataMan Setup Tool** 软件对康耐视扫码枪进行通信配置：

1. 将扫码枪的工业网络协议切换/启用为 **Modbus TCP** 模式 。
    
2. 确保扫码枪的 IP 地址与测试电脑处于**同一网段**（例如扫码枪：`192.168.1.100`，电脑：`192.168.1.10`） 。
    
3. 确认扫码枪的 Modbus 默认端口号（通常为 `502` 或是你自定义的映射端口） 。
    

### 2. 编译运行程序

1. 双击打开 `ScannerTool.sln` 解决方案 。
    
2. 打开 NuGet 包管理器，确认或安装 `NModbus` 依赖库 。
    
3. 点击 Visual Studio 的 **启动(Start)** 按钮编译并运行项目 。
    

### 3. 操作与调试说明

1. **输入参数**：在界面的文本框中分别输入扫码枪的 `IP 地址`（如 `192.168.1.100`）和 `网络端口` 。
    
2. **触发读码**：点击 **“触发读码”**（`btnTrigger`）按钮 ：
    
    - 软件会自动发起网络异步连接 。
        
    - 连接成功后发送 Modbus 触发拍照信号 。
        
    - 解析完成后，底部日志区域（`txtLogOutput`）将实时刷新测试结果 。
        
3. **返回状态解析**：
    
    - **【成功】**：输出解析出来的 ASCII 条码数据（如：`扫码数据 -> 690123456789`） 。
        
    - **【状态】**：如未对准条码或硬件超时，输出 `读码失败 (No Read)` 。
        
    - **【异常/错误】**：若中途断网或参数输入有误，系统会输出 `通信异常: 连接读码器超时` 等友好提示，同时自动恢复按钮可用状态，允许再次点击调试 。
        

## ⚙️ 核心Modbus映射参数参考

若后续需要对硬件做深度定制，可参考 `CognexScanner.cs` 中预设的核心常量 ：

|**参数名称**|**默认值**|**说明**|
|---|---|---|
|**StartAddress**|`2000`|输入寄存器起始读取地址|
|**TriggerCoil**|`0`|控制扫码枪拍照的线圈地址 (Write Single Coil)|
|**DefaultSlaveId**|`1`|Modbus 从站设备 ID|
|**DefaultTimeout**|`2000`|TCP 建立连接的超时断开时间 (单位: 毫秒)|
|**DecodeDelay**|`100`|发送触发后到读取数据之间的硬等待延迟 (单位: 毫秒)|
